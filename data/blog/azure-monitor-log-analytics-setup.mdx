## Overview

This case study explains how centralized monitoring and logging were implemented for the Azure infrastructure using **Azure Monitor** and **Log Analytics**.

The objective was to create centralized visibility into system activity, capture diagnostic logs, and enable administrators to analyze operational behavior across Azure resources.

This monitoring configuration is part of the larger **Secure Azure Storage Architecture** project.

---

## Monitoring Objectives

The monitoring architecture was designed to achieve the following goals:

- Enable centralized monitoring for Azure resources  
- Capture diagnostic logs for storage operations  
- Monitor system metrics and infrastructure activity  
- Detect abnormal behavior and operational issues  

---

## Azure Services Used

The following Azure services were used in this monitoring implementation:

- **Azure Monitor**
- **Log Analytics Workspace**
- **Azure Storage Account**
- **Diagnostic Settings**

---

# Step 1 — Create Log Analytics Workspace

A **Log Analytics Workspace** was created to collect and store monitoring data from Azure resources.

This workspace acts as the central repository where logs and metrics from different Azure services are aggregated and analyzed.

# Step 2 — Configure Diagnostic Settings

To capture operational activity from the storage account, **diagnostic settings** were configured.

These settings allow platform logs and metrics to be streamed directly to the **Log Analytics Workspace**.

![Diagnostic Settings Configuration](/static/images/azure-storage-diagnostic-settings-log-analytics-config.png)

Once the configuration was applied, the diagnostic settings confirmed that logs were successfully connected to the workspace.

![Diagnostic Settings Enabled](/static/images/azure-storage-diagnostic-settings-log-analytics-enabled.png)

This enables administrators to analyze storage activity and infrastructure behavior in near real-time.

---

## Log Analytics Integration

Logs from the storage account are continuously sent to the Log Analytics workspace.

Administrators can run queries using **Kusto Query Language (KQL)** to investigate infrastructure behavior and identify anomalies.

Examples of insights that can be detected include:

- unusual storage access patterns  
- abnormal request spikes  
- operational failures or errors  
- system performance issues

## Security and Operational Benefits

Implementing Azure Monitor and Log Analytics provides several operational and security advantages:

- centralized monitoring across Azure resources  
- improved troubleshooting and root-cause analysis  
- faster detection of abnormal activity  
- improved visibility into infrastructure operations
