- Role-Based Access Control (RBAC)
- Encryption at rest

## Architecture Design Principles

The architecture was designed based on key cloud security and reliability principles.

**Least Privilege Access**  
Access to resources is controlled using Role-Based Access Control (RBAC) to ensure users only have the permissions required for their roles.

**Network Isolation**  
Azure Virtual Networks and Network Security Groups are used to isolate the storage infrastructure and control inbound and outbound communication.

**Defense in Depth**  
Multiple layers of protection are implemented, including network security, RBAC permissions, encryption, and monitoring.

**Operational Visibility**  
Azure Monitor and Log Analytics provide continuous visibility into system behavior, enabling faster detection of unusual activity.

## Monitoring and Logging

To improve operational visibility, Azure Monitor diagnostic settings were enabled for the storage account.

Key configurations included:

- Diagnostic logs sent to Log Analytics
- Storage transaction metrics monitoring
- Centralized monitoring through Azure Monitor
- Query validation using Log Analytics Workspace

This setup improves visibility into storage operations and helps detect abnormal behavior.

## Alerting and Notifications

Azure Monitor alert rules were configured to proactively detect unusual activity.

Alert configuration included:

- Creation of an Azure Monitor Action Group
- Email notifications for operational alerts
- Alert rules for abnormal storage transaction spikes
- Automated alert triggering based on defined thresholds

This ensures administrators are notified quickly when suspicious or unexpected activity occurs.

## Security Validation

After deployment, the architecture was validated against Azure security best practices.

Validation checks included:

- Role-Based Access Control (RBAC) verification
- Network isolation through Azure Virtual Network
- Network Security Group rule enforcement
- Microsoft Defender for Cloud security recommendations

These controls help ensure the storage environment aligns with Azure cloud security and governance standards.

## Key Security Features

### Private Endpoints

Private Endpoints ensure that the storage account is only accessible through the Azure virtual network instead of broad public exposure.

### RBAC

Role-Based Access Control ensures that only authorized users are granted access to storage resources based on least-privilege principles.

### Encryption

Azure Storage automatically encrypts data at rest using Microsoft-managed keys to strengthen data protection.
