# Secure Azure Storage Architecture

In this project, I designed a secure Azure Storage architecture focusing on data protection and controlled access.

## Architecture Diagram

![Secure Azure Storage Architecture](/static/images/azure-secure-storage-architecture.png)

## Architecture Overview

The architecture includes:

- Azure Storage Account
- Azure Virtual Network (VNet)
- Private Endpoints
- Network Security Groups (NSG)
- Role-Based Access Control (RBAC)
- Encryption at rest

## Phase 1 - Storage Account Deployment

![Azure Storage Deployment Success](/static/images/storage-deployment-success.png)

The Azure Storage Account was successfully deployed in the CloudGuard resource group.  
This deployment confirms the secure storage infrastructure setup in the Azure environment before configuring private endpoints, RBAC access control, and encryption settings.

## Phase 2 — Network Security Configuration

![Azure Virtual Network Configuration](/static/images/vnet-success.png)

A virtual network was configured to isolate the storage infrastructure.  
Network Security Groups (NSGs) were applied to control inbound and outbound traffic.

![NSG Configuration](/static/images/nsg-overview.png)

Security rules were configured to restrict access and allow only required communication paths within the Azure environment.

## Key Security Features

### Private Endpoints
Private endpoints ensure that the storage account is only accessible within the Azure virtual network.

### RBAC
Role-based access control ensures only authorized users can access storage resources.

### Encryption
Azure Storage automatically encrypts data at rest using Microsoft-managed keys.

## Outcome

This architecture ensures:

- Secure data access
- Network isolation
- Controlled permissions
- Compliance with cloud security best practices

This project demonstrates practical experience designing secure cloud storage solutions using Microsoft Azure.
